# 国际化指南

v0plex 原生支持多语言站点。本页面介绍如何创建和维护多语言内容，以及国际化相关的配置和实现细节。

## 内容组织

多语言内容的核心组织原则是：每种语言独立一个目录，相同内容使用相同的文件名前缀。

### 目录结构

所有内容放在 `dev/` 目录下，按语言代码分子目录。如果你想支持中文、英文和日文，目录结构如下：

```
dev/
├── zh/
│   ├── 01_introduction.md
│   ├── 02_getting-started/
│   │   ├── 01_installation.md
│   │   └── 02_quick-start.md
│   └── 03_api/
│       └── 01_overview.md
├── en/
│   ├── 01_introduction.md
│   ├── 02_getting-started/
│   │   ├── 01_installation.md
│   │   └── 02_quick-start.md
│   └── 03_api/
│       └── 01_overview.md
└── ja/
    ├── 01_introduction.md
    ├── 02_getting-started/
    │   ├── 01_installation.md
    │   └── 02_quick-start.md
    └── 03_api/
        └── 01_overview.md
```

语言目录名必须与 `config/site.config.ts` 中 `AVAILABLE_LANGUAGES` 配置的 `folder` 字段匹配。

### 文件命名

文件名开头的数字前缀有两个作用。一是控制导航顺序，数字小的排在前面。二是用于跨语言匹配，相同前缀的文件会被识别为同一内容的不同语言版本。

`01_introduction.md` 和 `02_getting-started/` 目录会在导航中显示为"Introduction"和"Getting Started"，数字前缀会被移除。中文版的 `01_introduction.md` 和英文版的 `01_introduction.md` 会被识别为同一页面，用户可以切换语言跳转。

### 文件夹命名

文件夹同样使用数字前缀控制顺序。文件夹名称的前缀会被移除后作为分组名称显示在导航中。

## 语言配置

在 `config/site.config.ts` 中配置支持的语言。

### 添加新语言

在 `AVAILABLE_LANGUAGES` 数组中添加新的语言配置：

```typescript
AVAILABLE_LANGUAGES: [
  { code: 'zh', name: 'Chinese', nativeName: '中文', folder: 'zh' },
  { code: 'en', name: 'English', nativeName: 'English', folder: 'en' },
  { code: 'ja', name: 'Japanese', nativeName: '日本語', folder: 'ja' },
]
```

`code` 是语言代码，遵循 ISO 639-1 标准。`name` 是语言的英文名称。`nativeName` 是语言的本地名称，显示在语言选择器中。`folder` 是内容目录名。

添加新语言后，需要在 `dev/` 目录下创建对应的语言文件夹，并添加内容。还需要在 `I18N_STRINGS` 中添加翻译字符串。

### 设置默认语言

`DEFAULT_LOCALE` 决定用户首次访问时显示的语言：

```typescript
DEFAULT_LOCALE: 'zh'
```

如果用户浏览器中存储了语言偏好，会优先使用存储的偏好。默认语言只在用户首次访问且没有存储偏好时生效。

### 语言支持程度

`AVAILABLE_LANGUAGES_SUPPORT` 提供更详细的语言信息：

```typescript
AVAILABLE_LANGUAGES_SUPPORT: [
  {
    code: 'zh',
    name: 'Chinese',
    nativeName: '中文',
    folder: 'zh',
    support: 'full',
    progress: 100,
  },
  {
    code: 'en',
    name: 'English',
    nativeName: 'English',
    folder: 'en',
    support: 'partial',
    progress: 75,
  }
]
```

`support` 字段可以是 `full`（内容完整）或 `partial`（部分内容缺失）。`progress` 表示翻译进度百分比。这些信息可以在语言选择器中显示，让用户了解各语言的完成程度。

## 翻译字符串

`I18N_STRINGS` 定义各语言的 UI 文本翻译：

```typescript
I18N_STRINGS: {
  zh: {
    navigation: {
      previous: '上一篇',
      next: '下一篇',
    },
    lastUpdated: '最后更新于',
    editPage: '编辑此页',
    onThisPage: '本页目录',
    search: '搜索',
    language: '语言',
    theme: '主题',
    light: '浅色',
    dark: '深色',
    system: '跟随系统',
  },
  en: {
    navigation: {
      previous: 'Previous',
      next: 'Next',
    },
    lastUpdated: 'Last updated',
    editPage: 'Edit this page',
    onThisPage: 'On this page',
    search: 'Search',
    language: 'Language',
    theme: 'Theme',
    light: 'Light',
    dark: 'Dark',
    system: 'System',
  },
}
```

添加新语言时，需要添加对应的翻译字符串。如果某个字符串缺失，会回退到默认语言的字符串。

## 日期格式

不同地区有不同的日期格式偏好。`DATE_FORMATS` 为每种语言定义日期显示格式：

```typescript
DATE_FORMATS: {
  zh: 'yyyy年MM月dd日',
  en: 'MMMM d, yyyy',
  ja: 'yyyy年MM月dd日',
  de: 'dd.MM.yyyy',
}
```

格式字符串遵循 date-fns 库的规范。常用的格式标记包括 `yyyy` 四位年份、`yy` 两位年份、`MM` 两位月份、`MMM` 月份缩写、`MMMM` 月份全名、`dd` 两位日期、`d` 日期、`EEEE` 星期全名、`EEE` 星期缩写。

## 语言切换

v0plex 自动为每个页面生成跨语言链接。用户可以在任意页面切换到其他语言，会跳转到对应的内容页面。

### 链接生成原理

编译时，v0plex 会为每个页面计算 `prefixId`，即文件名的数字前缀。例如 `01_introduction.md` 的 `prefixId` 是 `01_`，`02_getting-started/01_installation.md` 的 `prefixId` 是 `02_01_`。

然后，v0plex 会匹配相同 `prefixId` 但不同语言的页面，生成 `languageLinks` 映射：

```typescript
{
  prefixId: '01_',
  languageLinks: {
    'zh': '/page/abc123',
    'en': '/page/def456',
    'ja': '/page/ghi789',
  }
}
```

前端根据这个映射渲染语言切换器。

### 处理缺失翻译

如果某个页面没有其他语言版本，语言切换器仍然显示所有可用语言，但点击后会跳转到该语言的首页或 404 页面。

你可以根据 `languageLinks` 是否包含对应语言，在 UI 上标识哪些语言有翻译。

## 内容翻译策略

### 完整翻译

最理想的情况是所有语言都有完整的内容。每个页面都翻译成所有支持的语言，导航结构一致。

这种情况下，用户在任何页面都能切换到任意语言，体验最好。

### 部分翻译

现实中常常无法做到完整翻译。可能有几种情况。某些页面只翻译了部分语言。某些语言的页面比其他语言少。某些页面的翻译不完整。

对于这种情况，可以在语言选择器中标注完成度。让用户知道哪些语言的内容更完整。

### 机器翻译辅助

使用机器翻译作为起点，然后人工校对修改，可以加速翻译过程。但需要注意机器翻译可能不准确，特别是技术术语和专业内容。

建议对机器翻译的内容进行标注，提醒读者可能存在不准确之处。

## 前端国际化实现

前端国际化主要涉及三个组件。

### LanguageProvider

`language-provider.tsx` 提供语言上下文。它读取 localStorage 中存储的语言偏好，如果没有则使用默认语言或浏览器语言。将当前语言注入 React 上下文，供其他组件使用。

### 导航数据处理

侧边栏根据当前语言加载对应的导航数据。导航数据存储在 `public/vmdjson/` 目录，按语言分开存储或在一个文件中包含语言信息。

### 翻译函数

组件中使用翻译函数获取本地化字符串：

```typescript
import { useLanguage } from '@/components/common/language-provider';

function MyComponent() {
  const { locale, t } = useLanguage();
  return <button>{t('search')}</button>;
}
```

`t` 函数根据当前语言返回对应的翻译字符串。

## 多语言 SEO

多语言站点需要考虑搜索引擎优化。

### hreflang 标签

每个页面应该包含指向其他语言版本的 `hreflang` 标签：

```html
<link rel="alternate" hreflang="zh" href="https://example.com/zh/page" />
<link rel="alternate" hreflang="en" href="https://example.com/en/page" />
```

这帮助搜索引擎理解页面的语言关系，向用户显示正确的语言版本。

### 语言特定 URL

v0plex 使用路径前缀区分语言，如 `/zh/`、`/en/`。这是 SEO 推荐的做法，每个语言版本有独立的 URL。

### 内容语言声明

HTML 文档应该声明内容语言：

```html
<html lang="zh">
```

v0plex 生成的页面会根据当前语言设置 `lang` 属性。

## 常见问题

### 语言切换后内容不匹配

确保相同内容的不同语言版本使用相同的文件名前缀。检查 frontmatter 中的标题是否正确翻译。

### 某些翻译字符串不显示

检查 `I18N_STRINGS` 中是否包含对应语言和键的翻译。检查组件中使用的键名是否正确。

### 日期格式显示错误

检查 `DATE_FORMATS` 中对应语言的格式字符串是否正确。确保 frontmatter 中的日期格式是 `YYYY-MM-DD`。

### 导航中文件夹名称未翻译

文件夹名称来自文件系统，v0plex 会移除数字前缀后直接使用。如果需要翻译文件夹名称，可以考虑在 frontmatter 或配置中定义映射。
