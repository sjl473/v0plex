# Directory Naming and i18n

## Directory Structure and Naming

For Markdown in various languages, all content is placed under the `dev/` directory, for example:

```
dev/
├── zh/
│   ├── _01_概述.md
│   ├── _02_快速开始/
│   │   ├── _01_安装.md
│   │   └── _02_快速上手.md
│   └── _03_API/
│       └── _01_概览.md
├── en/
│   ├── _01_overview.md
│   ├── _02_getting-started/
│   │   ├── _01_installation.md
│   │   └── _02_quick-start.md
│   └── _03_api/
│       └── _01_overview.md
```

**Mandatory requirement: All file and folder names must start with `_NN_`** (`NN` is a number), i.e., a numeric prefix with a leading underscore. For example, `_01_概述.md`, `_02_快速开始/`.

The numeric prefix at the beginning of the file name serves two purposes. First, it controls the navigation order; smaller numbers come first. Second, it is used for cross-language matching; **files with the same prefix are recognized as different language versions of the same content**.

`_01_概述.md` and `_02_快速开始/` will be displayed as "概述" and "快速开始" (Chinese) in the navigation, and as "Overview" and "Getting Started" in the English version; the numeric prefix will be removed. The Chinese `_01_概述.md` and the English `_01_overview.md` will be recognized as the same page, and users can switch languages to jump between them.

Below is another example containing a three-level directory structure:

```
dev/
├── zh/
│   ├── _01_概览.md
│   ├── _02_指南/
│   │   ├── _01_安装.md
│   │   ├── _02_配置/
│   │   │   ├── _01_基础配置.md
│   │   │   └── _02_高级配置.md
│   │   └── _03_部署.md
│   ├── _03_API/
│   │   ├── _01_接口.md
│   │   └── _02_认证.md
│   └── _04_附录/
│       └── _01_更新日志.md
└── en/
    ├── _01_overview.md
    ├── _02_guide/
    │   ├── _01_installation.md
    │   ├── _02_configuration/
    │   │   ├── _01_basic.md
    │   │   └── _02_advanced.md
    │   └── _03_deployment.md
    ├── _03_api/
    │   ├── _01_endpoints.md
    │   └── _02_authentication.md
    └── _04_appendix/

```

In this example:

- `_02_指南/` (English `_02_guide/`) is a group folder containing 3 files / subfolders
- `_02_配置/` (English `_02_configuration/`) is a sub-group inside `_02_指南/`, containing 2 files

## Common Errors in Multi-Level Directories

### 1. Inconsistent Numeric Prefixes Between Languages

This is the most common i18n error. If a certain `prefixId` exists in one language but is missing in another enabled language, the build will report error `E6006` (i18n prefix mismatch).

**Error Example**:

```
dev/
├── zh/
│   ├── _01_介绍.md
│   └── _02_指南/
│       └── _01_安装.md
└── en/
    ├── _01_intro.md
    └── _02_guide/
        └── _02_install.md   ← Error: should be _01_install.md, otherwise prefixId does not match
```

The `prefixId` of the English `_02_guide/_02_install.md` is `02_02`, while the corresponding position in the Chinese version is `02_01` (`prefixId` is `02_01`). Because the two `prefixId`s are different, they will not be recognized as the same page. At the same time, if `ENABLED_LANGUAGES` enables `en` and `zh`, the build will detect that the prefix `02_02` is missing in Chinese, triggering the `E6006` error.

**Correct approach**: Ensure that corresponding files / folders in each language directory structure use the **same numeric prefix**:

```
dev/
├── zh/
│   ├── _01_介绍.md
│   └── _02_指南/
│       └── _01_安装.md
└── en/
    ├── _01_intro.md
    └── _02_guide/
        └── _01_install.md   ← Correct: prefix matches the Chinese version
```

### 2. Missing a Hierarchy Causing prefixId Mismatch

If one language has fewer intermediate folders than another, even if the file's own numeric prefix is the same, the overall `prefixId` will be different.

**Error Example**:

```
dev/
├── zh/
│   └── _02_指南/
│       └── _01_安装.md       ← prefixId: 02_01
└── en/
    └── _01_install.md        ← prefixId: 01
```

The `prefixId`s of these two files are completely different and cannot be matched across languages.

**Correct approach**: Keep the hierarchy structure consistent:

```
dev/
├── zh/
│   └── _02_指南/
│       └── _01_安装.md
└── en/
    └── _02_guide/
        └── _01_install.md
```

### 3. Files or Folders Not Using the `_NN_` Format (Missing Leading Underscore)

v0plex **mandatorily requires** all Markdown files (`.md`, `.mdx`) and custom TSX files (`.tsx`), as well as all group folders, to be named using the `_NN_` format. If a file is named `01_概述.md` (without a leading underscore), the build will directly report error `E6007` (invalid prefix format).

**Error Example**:

```
dev/
├── zh/
│   ├── 01_介绍.md            ← Error: should be _01_介绍.md
│   └── _02_指南/
│       └── 01_安装.md        ← Error: should be _01_安装.md
```

**Correct approach**: All files and folders uniformly use the `_NN_` prefix:

```
dev/
├── zh/
│   ├── _01_介绍.md
│   └── _02_指南/
│       └── _01_安装.md
```

### 4. Enabled a Language with No Configured Content

A language is enabled in `ENABLED_LANGUAGES`, but the `dev/` directory has no folder or files for that language at all. The build will trigger errors such as `V2001`, `V2004`, or `E6005`.

**Example**: `ENABLED_LANGUAGES: ['en', 'zh', 'ja']`, but `dev/` only has `en/` and `zh/`, missing `ja/`.

### 6. Language Folder Name Inconsistent with Configuration

The language folder names under `dev/` must match the `folder` field defined in `AVAILABLE_LANGUAGES_SUPPORT`. For example, if the `folder` field for Chinese in the configuration is `'zh'`, the folder name must be `zh/`, not `cn/` or `chinese/`.
