# 错误码参考（kimi-k2.6 生成）

**前缀说明：**

- `E` 开头：通用错误（E1xxx 系统级、E2xxx frontmatter, E3xxx Markdown, E4xxx 扩展语法、E5xxx 资源处理、E6xxx 构建、E7xxx 文件系统、E8xxx 作者格式）
- `V` 开头：v0plex 专用错误（V0xxx 配置、V1xxx Git 操作、V2xxx dev 目录、V3xxx 语言配置）


## 系统级错误 (E1xxx)

| 错误码 | 说明 | 触发原因与解决方法 |
|--------|------|--------------------|
| `E1000` | 未知错误 | 发生了未预期的异常。查看错误详情中的 `details` 字段。|
| `E1001` | 配置错误 | `site.config.ts` 或其他配置文件存在错误。根据提示修复配置。|
| `E1002` | 文件系统错误 | 无法访问某个文件或目录。检查文件权限和路径是否正确。|

### 示例

- `E1001` — `config/site.config.ts` 中缺少括号：
  ```typescript
  export const BUILD_CONFIG = {
    PUBLIC_DIR: 'public',
    // 缺少闭合的 };
  ```
- `E1002` — 尝试读取没有权限的文件，或磁盘已损坏导致无法访问。


## 配置错误 (V0xxx)

| 错误码 | 说明 | 触发原因与解决方法 |
|--------|------|--------------------|
| `V0001` | 配置解析失败 | `config/site.config.ts` 中存在语法错误或无法解析的内容。检查 TypeScript 语法。|
| `V0002` | 缺少仓库地址 | 远程模式下（`USE_LOCAL_MARKDOWN: false`），必须在 `GIT_CONFIG.REPO_URL` 中指定 Git 仓库地址。|
| `V0003` | 分支名称无效 | `GIT_CONFIG.BRANCH` 中包含非法字符。分支名只能包含字母、数字、连字符、下划线、点和斜杠。|

### 示例

- `V0002` — 远程模式但未配置仓库：
  ```typescript
  export const CONTENT_SOURCE_CONFIG = {
    USE_LOCAL_MARKDOWN: false,
    GIT_CONFIG: {
      REPO_URL: '', // 空值会触发 V0002
      BRANCH: 'main',
    },
  };
  ```
- `V0003` — 分支名包含中文：
  ```typescript
  BRANCH: 'feat/新功能'
  ```


## Git 操作错误 (V1xxx)

| 错误码 | 说明 | 触发原因与解决方法 |
|--------|------|--------------------|
| `V1000` | 克隆仓库失败 | 无法从配置的 `REPO_URL` 克隆仓库。检查网络连接、仓库地址和访问权限。|
| `V1001` | 检出提交失败 | 无法切换到指定的 commit。确认分支或 commit hash 正确。|
| `V1002` | 删除 dev 目录失败 | 清理旧 dev 目录时出错。检查目录权限，或手动删除 `dev/` 后重试。|
| `V1003` | Git 不可用 | 系统中未安装 Git，或 Git 不在 `PATH` 环境变量中。安装 Git 并确保命令行可用。|

### 示例

- `V1000` — 仓库地址错误或网络不通：
  ```typescript
  REPO_URL: 'https://github.com/sjl473/不存在的仓库'
  ```
- `V1001` — 分支名拼写错误导致无法检出：
  ```typescript
  BRANCH: 'mian' // 应为 'main'
  ```
- `V1003` — 在全新容器中运行构建，但未安装 Git：
  ```bash
  $ git --version
  zsh: command not found: git
  ```


## dev 目录错误 (V2xxx)

| 错误码 | 说明 | 触发原因与解决方法 |
|--------|------|--------------------|
| `V2000` | dev 目录不存在 | 本地模式下 `dev/` 目录不存在。创建 `dev/` 目录并添加内容文件。|
| `V2001` | dev 目录缺少语言文件夹 | `dev/` 下必须至少有一个语言文件夹（如 `en`、`zh`），对应 `ENABLED_LANGUAGES` 中的配置。|
| `V2002` | dev 目录没有文件 | `dev/` 目录下全是空文件夹，没有任何文件。请添加 Markdown 文件。|
| `V2003` | dev 目录结构无效 | 远程模式要求 `dev/{language}/...` 的结构。检查语言文件夹命名是否正确。|
| `V2004` | 缺少必需的语言文件夹 | `ENABLED_LANGUAGES` 中启用了某些语言，但 `dev/` 下缺少对应的语言文件夹。|
| `V2005` | 存在未配置的语言文件夹 | `dev/` 下有不在 `ENABLED_LANGUAGES` 中的语言文件夹。删除多余文件夹或将其加入配置。|

### 示例

- `V2000` — 本地模式下项目根目录没有 `dev/`：
  ```
  v0plex/
  ├── app/
  ├── config/
  └── public/
  ```
- `V2001` — `dev/` 下直接放文件，没有语言子目录：
  ```
  dev/
  └── hello.md   ← 错误：应该放在 dev/zh/hello.md
  ```
- `V2002` — `dev/zh/` 是一个空文件夹，没有任何 Markdown 文件。
- `V2004` — `ENABLED_LANGUAGES` 配置了 `['en', 'zh']`，但 `dev/` 下只有：
  ```
  dev/
  └── en/
  ```
- `V2005` — `ENABLED_LANGUAGES` 只有 `['en']`，但 `dev/` 下存在：
  ```
  dev/
  ├── en/
  └── zh/   ← 多余的语言文件夹
  ```

## Frontmatter 错误 (E2xxx)

| 错误码 | 说明 | 触发原因与解决方法 |
|--------|------|--------------------|
| `E2000` | Frontmatter 解析失败 | Markdown 文件顶部的 `---...---` 块无法被解析。检查是否有格式问题，如缺少结束 `---`。|
| `E2001` | 缺少必需的 frontmatter 字段 | 缺少 `title`、`created_at`、`last_updated_at`、`author`、`has_custom_tsx`、`tags` 中的一个或多个。补全缺失字段。|
| `E2002` | Frontmatter 字段值无效 | 某个字段的值格式不正确。例如 `has_custom_tsx` 不是 `'true'` 或 `'false'`；日期不是 `YYYY-MM-DD` 或 `@git`。|
| `E2003` | Frontmatter 格式错误 | 包含了不允许的额外字段。只允许 `title`、`created_at`、`last_updated_at`、`author`、`has_custom_tsx`、`tags` 这六个字段。|
| `E2004` | 作者格式无效 | `author` 字段的格式不符合要求。请使用 `名字`、`名字->邮箱`、`名字->URL` 或 `@git`，多个作者用 `\|` 分隔。|

### 示例

- `E2000` — 缺少 frontmatter 结束标记：
  ```markdown
  ---
  title: 测试
  created_at: 2024-01-01
  <!-- 缺少 --- 结束标记 -->
  # 正文
  ```
- `E2001` — 缺少必填字段：
  ```yaml
  ---
  title: 测试
  created_at: 2024-01-01
  ---
  ```
  缺少 `last_updated_at`、`author`、`has_custom_tsx`、`tags`。
- `E2002` — `has_custom_tsx` 使用了布尔值而非字符串：
  ```yaml
  has_custom_tsx: false   ← 错误
  has_custom_tsx: 'false' ← 正确
  ```
- `E2003` — 包含不允许的额外字段：
  ```yaml
  ---
  title: 测试
  created_at: 2024-01-01
  last_updated_at: 2024-01-01
  author: a->a@b.com
  tags: [test]
  has_custom_tsx: 'false'
  category: guide   ← 不允许的字段
  ---
  ```
- `E2004` — 作者格式使用了尖括号：
  ```yaml
  author: 张三 <zhangsan@example.com>   ← 错误
  author: 张三->zhangsan@example.com    ← 正确
  ```
- `E2004` — 作者格式支持邮箱或 URL：
  ```yaml
  author: '张三->zhangsan@example.com | 李四->https://lisi.dev'
  ```
- `E2004` — 使用了双箭头（不支持）：
  ```yaml
  author: '张三->zhangsan@example.com->https://zhangsan.dev'   ← 错误：只能有一个 ->
  ```


## Markdown 编译错误 (E3xxx)

| 错误码 | 说明 | 触发原因与解决方法 |
|--------|------|--------------------|
| `E3000` | Markdown 编译失败 | 编译过程中发生了未预期的错误。查看错误详情定位具体原因。|
| `E3001` | Markdown 语法错误 | Markdown 源码存在语法问题。根据提示的行号检查内容。|
| `E3002` | 嵌套代码块 | 在一个围栏代码块（```）内部又出现了一个新的围栏代码块。代码块不能嵌套。检查是否遗漏了结束标记。|
| `E3003` | 检测到 XSS 攻击模式 | 数学公式中包含了 `<script>`、`javascript:`、`onload=` 等危险内容。清理公式中的恶意代码。|
| `E3004` | 检测到空标记元素 | 输出中存在空的加粗、斜体、删除线或行内代码元素。检查 Markdown 中是否有 `****`、```` 等空标记。|
| `E3005` | 单行反引号代码块 | 在行首使用单引号 `` ` `` 作为代码块分隔符。VMD 只允许使用三引号 `` ``` `` 作为围栏代码块。|
| `E3006` | 无效的标签嵌套 | `<post>`、`<lft>`、`<rt>` 被错误地嵌套在 `<info>`、`<warning>`、`<success>`、`<smallimg>` 内部，或者 `<lft>`/`<rt>` 出现在 `<post>` 外部。调整标签层级。|

### 示例

- `E3002` — 代码块没有正确闭合导致嵌套：
  ````markdown
  ```js
  function a() {
    return 1;
  }
  ```ts      ← 结束标记用了 ```ts 而非 ```，导致下一个 ``` 被误认为嵌套
  const x = 1;
  ```
  ````
- `E3003` — 数学公式中注入脚本：
  ```markdown
  $<script>alert(1)</script>$
  ```
- `E3004` — 空的 Markdown 标记：
  ```markdown
  ****

  ~~ ~~

  ```` `` ````
  ```
- `E3005` — 在行首用单引号作为代码块（VMD 不允许）：
  ```markdown
  `this is not inline code but at line start
  ```
- `E3006` — `<post>` 嵌套在 `<info>` 内部：
  ```markdown
  <info>
  <post>
  <lft>

  内容

  </lft>
  <rt>

  ![图](a.png)

  </rt>
  </post>
  </info>
  ```


## 扩展语法错误 — `<post>` 布局 (E4xxx)

| 错误码 | 说明 | 触发原因与解决方法 |
|--------|------|--------------------|
| `E4000` | `<post>` 缺少 `<lft>` 或 `<rt>` | `<post>` 块中必须同时包含一个 `<lft>` 和一个 `<rt>` 标签。检查标签是否齐全。|
| `E4001` | `<post>` `<rt>` 内容无效 | `<rt>` 中只能包含图片，不能包含其他文字；图片必须单独成行；图片之间必须有换行分隔；图片必须有 alt 文本。|
| `E4002` | `<post>` `<rt>` 缺少图片 | `<rt>` 中必须至少有一张图片。在 `<rt>` 中添加 `![alt](src)` 格式的图片。|
| `E4003` | `<post>` 图片缺少 alt 文本 | `<rt>` 中的图片必须有 alt 文本。使用 `![描述](图片路径)` 的格式。|
| `E4004` | 自定义块处理失败 | `<info>`、`<warning>` 或 `<success>` 等自定义块处理时出错。查看具体错误信息。|
| `E4005` | `<post>` 格式错误 | `<post>` 与 `<lft>` 之间、`</lft>` 与 `<rt>` 之间、`</rt>` 与 `</post>` 之间必须只有一个空格或换行符。调整标签间距。|
| `E4006` | `<lft>` 包含无效 HTML 标签 | `<lft>` 中只能使用行内元素，不允许使用块级 HTML 标签。|
| `E4007` | `<post>` 间距错误 | `<lft>` 或 `<rt>` 内容的开始和结束处必须有双换行符（即内容前后各空一行）；`<rt>` 中必须包含图片。|
| `E4008` | `<post>` 中包含代码块 | `<lft>` 中不允许使用围栏代码块（```）。如果需要展示代码，请放在 `<post>` 外部。|
| `E4009` | `<post>` 中包含禁止的标签 | `<lft>` 中不允许包含 `<post>`、`<lft>`、`<rt>`、`<info>`、`<warning>`、`<success>`、`<smallimg>` 标签。|
| `E4010` | `<post>` 中包含块级数学公式 | `<lft>` 中不允许使用 `$$...$$` 块级数学公式。请使用行内数学公式 `$...$`，或将其放在 `<post>` 外部。|
| `E4011` | `<lft>` 中包含普通图片 | `<lft>` 中不允许使用 `![alt](src)` 普通 Markdown 图片。如需小图请使用 `<smallimg>![alt](src)</smallimg>`。|
| `E4012` | `<post>` 中包含引用块 | `<lft>` 中不允许使用 `>` 开头的引用块。请将其放在 `<post>` 外部。|
| `E4013` | `<post>` 间距重复错误 | `<post>` 块内部的换行间距不符合要求。确保 `<lft>` 和 `<rt>` 的内容前后都有空行。|
| `E4014` | 空自定义块 | `<info>`、`<warning>` 或 `<success>` 标签内没有任何内容。请添加内容或删除空块。|

### 示例

- `E4000` — 缺少 `<rt>`：
  ```markdown
  <post>
  <lft>

  只有左侧内容

  </lft>
  </post>
  ```
- `E4001` — `<rt>` 中包含非图片文字：
  ```markdown
  <rt>

  这是文字       ← 错误
  ![图](a.png)

  </rt>
  ```
- `E4002` — `<rt>` 中没有任何图片：
  ```markdown
  <rt>



  </rt>
  ```
- `E4003` — 图片缺少 alt：
  ```markdown
  <rt>

  ![](a.png)     ← 错误：alt 为空

  </rt>
  ```
- `E4005` — 标签间距不正确：
  ```markdown
  <post>

  <lft>          ← 错误：<post> 和 <lft> 之间有两个换行
  ```
- `E4007` — 内容前后没有空行：
  ```markdown
  <lft>
  内容           ← 错误：内容前没有空行
  </lft>
  ```
- `E4008` — `<lft>` 中包含代码块：
  ```markdown
  <lft>

  ```js
  const x = 1;
  ```

  </lft>
  ```
- `E4009` — `<lft>` 中嵌套 `<info>`：
  ```markdown
  <lft>

  <info>提示</info>

  </lft>
  ```
- `E4010` — `<lft>` 中使用块级数学：
  ```markdown
  <lft>

  $$E = mc^2$$

  </lft>
  ```
- `E4011` — `<lft>` 中使用普通图片：
  ```markdown
  <lft>

  ![截图](ui.png)

  </lft>
  ```
- `E4012` — `<lft>` 中使用引用块：
  ```markdown
  <lft>

  > 这是一段引用

  </lft>
  ```
- `E4014` — 空的提示框：
  ```markdown
  <info>
  </info>
  ```


## 表格错误 (E41xx)

| 错误码 | 说明 | 触发原因与解决方法 |
|--------|------|--------------------|
| `E4100` | 表格内容无效 | 表格的整体结构无法被正确解析。检查 `<table>` 包裹的内容是否完整。|
| `E4101` | 表格中包含块级元素 | 表格单元格内不允许出现 `<div>`、`<p>`、`<h1>` 等块级 HTML 标签。只使用行内格式。|
| `E4102` | 表格中包含图片 | 表格单元格内不允许使用 `![alt](src)` 图片。请从单元格中移除图片。|
| `E4103` | 表格中包含列表 | 表格单元格内不允许使用 `-`、`*`、`+` 或 `1.` 开头的列表项。|
| `E4104` | 表格中包含引用块 | 表格单元格内不允许使用 `>` 开头的引用内容。|
| `E4105` | 表格中包含代码块 | 表格单元格内不允许使用围栏代码块（```）。只使用行内代码 `` `code` ``。|
| `E4106` | 表格中包含块级数学公式 | 表格单元格内不允许使用 `$$...$$`。只使用行内数学公式 `$...$`。|
| `E4107` | 表格格式无效 | 分隔行（`|---|---|`）格式不正确。确保分隔行只包含 `-`、`|`、`:` 和空格。|

### 示例

- `E4100` — 表格结构不完整（缺少分隔行）：
  ```markdown
  <table>

  | 名称 |
  | 值   |

  </table>
  ```
- `E4101` — 单元格中使用块级 HTML：
  ```markdown
  <table>

  | 内容 |
  |------|
  | <div>块级元素</div> |

  </table>
  ```
- `E4102` — 单元格中包含图片：
  ```markdown
  | 图标 |
  |------|
  | ![图标](icon.png) |
  ```
- `E4103` — 单元格中包含列表：
  ```markdown
  | 项目 |
  |------|
  | - 第一项 |
  ```
- `E4104` — 单元格中包含引用：
  ```markdown
  | 说明 |
  |------|
  | > 注意事项 |
  ```
- `E4105` — 单元格中包含代码块：
  ```markdown
  | 代码 |
  |------|
  | ```js const x = 1; ``` |
  ```
- `E4106` — 单元格中包含块级数学：
  ```markdown
  | 公式 |
  |------|
  | $$x^2$$ |
  ```
- `E4107` — 分隔行包含非法字符：
  ```markdown
  | 名称 | 类型 |
  | --#- | --%- |   ← 包含 # 和 %
  | id   | number |
  ```


## 资源处理错误 (E5xxx)

| 错误码 | 说明 | 触发原因与解决方法 |
|--------|------|--------------------|
| `E5000` | 资源处理错误 | 处理图片或代码文件时发生了通用错误。查看 `message` 详情。|
| `E5001` | 图片未找到 | 引用的图片文件在搜索路径中不存在。搜索路径包括：当前文件目录、`dev/assets/images/`、`public/`。确认图片路径正确且文件存在。也禁止使用远程图片 URL（`http://` 或 `https://`）。|
| `E5002` | 图片处理失败 | 读取或复制图片文件时出错。检查文件权限和图片文件是否损坏。|
| `E5003` | 代码文件写入失败 | 无法将提取的代码写入 `public/vmdcode/` 目录。检查目录权限和磁盘空间。|

### 示例

- `E5001` — 引用不存在的本地图片：
  ```markdown
  ![截图](screenshot.png)
  ```
  但 `screenshot.png` 不在当前目录、`dev/assets/images/` 或 `public/` 中。
- `E5001` — 使用远程图片（被禁止）：
  ```markdown
  ![头像](https://example.com/avatar.png)
  ```
- `E5002` — 图片文件已损坏，或当前用户没有读取该文件的权限。
- `E5003` — `public/vmdcode/` 目录只读，导致无法写入提取的代码文件。


## 构建错误 (E6xxx)

| 错误码 | 说明 | 触发原因与解决方法 |
|--------|------|--------------------|
| `E6000` | 构建错误 | 整体构建过程失败。查看前面的错误信息了解具体原因。|
| `E6001` | 缺少一级标题 | Markdown 正文必须以 `# 标题` 开头。在正文第一行添加一个一级标题。|
| `E6002` | 缺少自定义 TSX 文件 | `has_custom_tsx: 'true'` 时，必须存在与 Markdown 文件同名（扩展名改为 `.tsx`）的 TSX 文件。创建对应的 TSX 组件文件。|
| `E6003` | 无效的构建路径 | 输入路径不存在或无法访问。检查 `dev/` 目录路径是否正确。|
| `E6004` | Git 不可用 | 文件中包含 `@git` 占位符，但系统未安装 Git 或无法访问。安装 Git 或移除 `@git` 占位符。|
| `E6005` | 无效的语言文件夹 | `dev/` 下存在不属于 `ENABLED_LANGUAGES` 配置的文件夹。删除无效文件夹或将其加入语言配置。|
| `E6006` | i18n 前缀不匹配 | 同一个内容前缀在某些语言中存在，但在其他启用的语言中缺失。确保所有启用的语言都有对应的文件结构。|
| `E6007` | 文件 / 文件夹名格式无效 | Markdown 文件、TSX 文件或分组文件夹未使用 `_NN_` 格式（缺少前导下划线）。将所有文件和文件夹重命名为 `_NN_名称` 格式。|

### 示例

- `E6001` — 正文开头不是一级标题：
  ```markdown
  ## 二级标题     ← 错误：必须以 # 开头
  正文内容
  ```
- `E6002` — 启用自定义 TSX 但缺少文件：
  ```
  dev/
  └── zh/
      ├── _01_my-page.md      ← has_custom_tsx: 'true'
      └── _01_my-page.tsx     ← 缺少此文件会触发 E6002
  ```
- `E6004` — 使用 `@git` 但 Git 未安装：
  ```yaml
  ---
  created_at: @git
  ---
  ```
- `E6006` — 多语言内容缺失：
  ```
  dev/
  ├── en/
  │   └── _01_start.md
  └── zh/
      └── _02_guide.md   ← 缺少 _01_start.md 的对应文件
  ```
- `E6007` — 文件或文件夹名缺少前导下划线（以下以中文 `zh` 和英文 `en` 两种语言的 `dev` 文件夹为例）：
  ```
  dev/
  ├── zh/
  │   ├── 01_intro.md              ← 错误：应为 _01_intro.md
  │   └── _02_guide/
  │       └── 01_install.md        ← 错误：应为 _01_install.md
  └── en/
      ├── _01_intro.md             ← 正确
      └── _02_guide/
          └── _01_install.md       ← 正确
  ```
  中文 `zh` 目录下的 `01_intro.md` 和 `01_install.md` 缺少前导下划线，与英文 `en` 目录中对应文件 `_01_intro.md`、`_01_install.md` 的命名格式不一致，会触发 `E6007`。

## 文件系统错误 (E7xxx)

| 错误码 | 说明 | 触发原因与解决方法 |
|--------|------|--------------------|
| `E7000` | 清理目录失败 | 清理构建输出目录时出错。检查目录权限。|
| `E7001` | 删除文件失败 | 无法删除某个文件。检查文件是否被其他程序占用，或权限不足。|
| `E7002` | 创建目录失败 | 无法创建输出目录。检查父目录的写入权限。|
| `E7003` | 重置目录失败 | 清空目录内容时出错。检查目录权限和磁盘状态。|
| `E7004` | 读取文件失败 | 无法读取源文件。检查文件是否存在且具有读取权限。|
| `E7005` | 写入文件失败 | 无法写入输出文件。检查目标目录的写入权限和磁盘空间。|
| `E7006` | 写入站点数据失败 | 无法写入 `site-data.json`。检查 `public/vmdjson/` 目录权限。|

### 示例

- `E7004` — 编译过程中源文件被其他程序删除或锁定。
- `E7005` — 磁盘已满导致无法写入生成的 `page.tsx`：
  ```bash
  Error: ENOSPC: no space left on device
  ```
- `E7006` — `public/vmdjson/` 目录权限为只读，无法写入 `site-data.json`。

## 作者格式错误 (E8xxx)

| 错误码 | 说明 | 触发原因与解决方法 |
|--------|------|--------------------|
| `E8000` | 作者列表为空 | `author` 字段值为空，且不是 `@git`。请填写作者信息。|
| `E8001` | 多个 `@git` 占位符 | `author` 字段中包含多个 `@git`，最多只允许一个。保留一个 `@git` 即可。|
| `E8002` | 作者条目格式无效 | 某个作者条目格式不正确。请使用 `名字`、`名字->邮箱`、`名字->URL` 或 `@git`。|
| `E8003` | 作者名称为空 | 作者名称不能为空字符串。请提供有效的作者名称。|
| `E8004` | 作者邮箱重复 | 多个作者条目使用了相同的邮箱地址。请确保每个作者的邮箱唯一。|
| `E8005` | 作者 URL 重复 | 多个作者条目使用了相同的 URL。请确保每个作者的 URL 唯一。|
| `E8006` | 邮箱或 URL 格式无效 | `名字->邮箱` 中的邮箱格式不正确，或 URL 格式不正确。请使用标准的邮箱或 URL 格式。|
| `E8007` | 作者条目为空 | `author` 字段用 `\|` 分隔后存在空字符串。检查 `\|` 两侧是否有空白或多余的 `\|`。|

### 示例

- `E8000` — 作者字段为空：
  ```yaml
  author: ''
  ```
- `E8001` — 多个 `@git`：
  ```yaml
  author: '@git | @git'
  ```
- `E8002` — 箭头后缺少内容：
  ```yaml
  author: '张三->'   ← 邮箱或 URL 为空
  ```
- `E8002` — 使用了双箭头（不支持）：
  ```yaml
  author: '张三->a@b.com->https://example.com'   ← 错误：只能有一个 ->
  ```
- `E8003` — 名称缺失：
  ```yaml
  author: '->email@test.com'
  ```
- `E8004` — 邮箱重复：
  ```yaml
  author: '张三->a@b.com | 李四->a@b.com'
  ```
- `E8006` — 无效的邮箱或 URL 格式：
  ```yaml
  author: '张三->not-an-email'      ← 错误：不是有效邮箱
  author: '张三->example.com'       ← 错误：URL 缺少 http:// 或 https:// 前缀
  ```
- `E8007` — 多余的 `|` 导致空条目：
  ```yaml
  author: '张三 | | 李四'
  ```
