# 目录命名和 i18n

## 目录结构和命名

对于各种语言的 Markdown，所有内容放在 `dev/` 目录下，比如：

```
dev/
├── zh/
│   ├── _01_概述.md
│   ├── _02_快速开始/
│   │   ├── _01_安装.md
│   │   └── _02_快速上手.md
│   └── _03_API/
│       └── _01_概览.md
├── en/
│   ├── _01_overview.md
│   ├── _02_getting-started/
│   │   ├── _01_installation.md
│   │   └── _02_quick-start.md
│   └── _03_api/
│       └── _01_overview.md
```

**强制要求：所有文件和文件夹名必须以 `_NN_` 开头**（`NN` 为数字），即带前导下划线的数字前缀。例如 `_01_概述.md`、`_02_快速开始/`。

文件名开头的数字前缀有两个作用。一是控制导航顺序，数字小的排在前面。二是用于跨语言匹配，**相同前缀的文件会被识别为同一内容的不同语言版本**。

`_01_概述.md` 和 `_02_快速开始/` 目录会在导航中显示为 "概述" 和 "快速开始"（中文），英文版则显示为 "Overview" 和 "Getting Started"，数字前缀会被移除。中文版的 `_01_概述.md` 和英文版的 `_01_overview.md` 会被识别为同一页面，用户可以切换语言跳转。

下面是一个包含三层目录结构的另一个示例：

```
dev/
├── zh/
│   ├── _01_概览.md
│   ├── _02_指南/
│   │   ├── _01_安装.md
│   │   ├── _02_配置/
│   │   │   ├── _01_基础配置.md
│   │   │   └── _02_高级配置.md
│   │   └── _03_部署.md
│   ├── _03_API/
│   │   ├── _01_接口.md
│   │   └── _02_认证.md
│   └── _04_附录/
│       └── _01_更新日志.md
└── en/
    ├── _01_overview.md
    ├── _02_guide/
    │   ├── _01_installation.md
    │   ├── _02_configuration/
    │   │   ├── _01_basic.md
    │   │   └── _02_advanced.md
    │   └── _03_deployment.md
    ├── _03_api/
    │   ├── _01_endpoints.md
    │   └── _02_authentication.md
    └── _04_appendix/
        └── _01_changelog.md
```

在这个例子中：

- `_02_指南/`（英文 `_02_guide/`）是一个分组文件夹，内部包含 3 个文件 / 子文件夹
- `_02_配置/`（英文 `_02_configuration/`）是 `_02_指南/` 内部的子分组，包含 2 个文件

## 多层目录的常见错误

### 1. 不同语言之间数字前缀不一致

这是最常见的 i18n 错误。如果某个 `prefixId` 在一种语言中存在，但在另一种启用的语言中缺失，构建时会报错 `E6006`（i18n prefix mismatch）。

**错误示例**：

```
dev/
├── zh/
│   ├── _01_介绍.md
│   └── _02_指南/
│       └── _01_安装.md
└── en/
    ├── _01_intro.md
    └── _02_guide/
        └── _02_install.md   ← 错误：应为 _01_install.md，否则 prefixId 不匹配
```

英文版 `_02_guide/_02_install.md` 的 `prefixId` 是 `02_02`，而中文版对应位置是 `02_01`（`prefixId` 为 `02_01`）。由于两者 `prefixId` 不同，不会被识别为同一页面。同时，如果 `ENABLED_LANGUAGES` 启用了 `en` 和 `zh`，构建时会检测到 `02_02` 这个前缀在中文中缺失，触发 `E6006` 错误。

**正确做法**：确保各语言目录结构中，对应位置的文件 / 文件夹使用 **相同的数字前缀**：

```
dev/
├── zh/
│   ├── _01_介绍.md
│   └── _02_指南/
│       └── _01_安装.md
└── en/
    ├── _01_intro.md
    └── _02_guide/
        └── _01_install.md   ← 正确：与中文版前缀一致
```

### 2. 缺少某一层级导致 `prefixId` 对不上

如果一种语言比另一种语言少了一个中间文件夹，即使文件本身的数字前缀相同，整体 `prefixId` 也会不同。

**错误示例**：

```
dev/
├── zh/
│   └── _02_指南/
│       └── _01_安装.md       ← prefixId: 02_01
└── en/
    └── _01_install.md        ← prefixId: 01
```

这两个文件的 `prefixId` 完全不同，无法跨语言匹配。

**正确做法**：保持层级结构一致：

```
dev/
├── zh/
│   └── _02_指南/
│       └── _01_安装.md
└── en/
    └── _02_guide/
        └── _01_install.md
```

### 3. 文件或文件夹未使用 `_NN_` 格式（缺少前导下划线）

v0plex **强制要求** 所有 Markdown 文件（`.md`、`.mdx`）和自定义 TSX 文件（`.tsx`），以及所有分组文件夹，都必须使用 `_NN_` 格式命名。如果文件名写成了 `01_概述.md`（没有前导下划线），构建时会直接报错 `E6007`（invalid prefix format）。

**错误示例**：

```
dev/
├── zh/
│   ├── 01_介绍.md            ← 错误：应为 _01_介绍.md
│   └── _02_指南/
│       └── 01_安装.md        ← 错误：应为 _01_安装.md
```

**正确做法**：所有文件和文件夹统一使用 `_NN_` 前缀：

```
dev/
├── zh/
│   ├── _01_介绍.md
│   └── _02_指南/
│       └── _01_安装.md
```

### 4. 启用了未配置内容的语言

`ENABLED_LANGUAGES` 中启用了某种语言，但 `dev/` 下完全没有该语言的文件夹或文件。构建时会触发 `V2001`、`V2004` 或 `E6005` 等错误。

**示例**：`ENABLED_LANGUAGES: ['en', 'zh', 'ja']`，但 `dev/` 下只有 `en/` 和 `zh/`，缺少 `ja/`。

### 6. 语言文件夹命名与配置不一致

`dev/` 下的语言文件夹名称必须与 `AVAILABLE_LANGUAGES_SUPPORT` 中定义的 `folder` 字段一致。例如，如果配置中中文的 `folder` 是 `'zh'`，那么文件夹名必须是 `zh/`，不能是 `cn/` 或 `chinese/`。
