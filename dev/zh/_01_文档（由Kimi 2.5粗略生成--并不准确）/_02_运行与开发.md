# 运行与开发

完成安装后，即可开始使用 v0plex 进行内容创作。本章介绍开发服务器的使用、构建流程的细节，以及内容组织的基本原则。

## 启动开发服务器

v0plex 提供热重载的开发服务器，可以在编辑内容时实时预览效果。在项目根目录执行：

```bash
pnpm dev
```

这个命令实际上是三个步骤的组合：首先执行 `pnpm vmd:gen` 解析 Markdown 文件，然后执行 `pnpm lex:gen` 生成中文搜索索引，最后启动 Next.js 开发服务器。首次启动时，构建过程可能需要 30 秒到 2 分钟，取决于内容的数量和系统的性能。

开发服务器默认监听 `0.0.0.0:3000`，这意味着它不仅可以在本地通过 `http://localhost:3000` 访问，也可以通过网络中的其他设备访问，只要防火墙允许。如果你需要指定其他端口，可以修改 `package.json` 中的 `dev` 脚本，添加 `-p` 参数。

## 内容编辑工作流

v0plex 的设计理念是将内容创作与技术实现分离。所有内容都位于 `dev/` 目录中，你可以直接在这里创建、编辑和删除 Markdown 文件。

文件组织遵循特定的命名规范。文件夹和文件以 `_NN_` 前缀开头，`NN` 是两位数字，控制内容在导航树中的显示顺序。例如，`_01_快速开始/` 会排在 `_02_高级主题/` 之前。这种命名方式使得排序不依赖文件系统的时间戳或字母顺序，而是由作者明确控制。

在编辑 Markdown 文件时，注意 frontmatter 的格式要求。每个文件的开头必须包含 YAML 格式的元数据块，用三个短横线 `---` 包围。必需的字段包括 `title`（显示标题）、`created_at`（创建日期）、`last_updated_at`（最后更新日期）和 `author`（作者）。可选字段 `has_custom_tsx` 用于标记该文档是否关联了自定义 TSX 组件。

当你保存修改后，开发服务器会自动重新构建受影响的部分。但有时自动重载可能不完全生效，特别是修改了 frontmatter 或文件结构时。这种情况下，按 Ctrl+C 停止服务器，然后重新执行 `pnpm dev` 即可。

## 添加图像资源

技术文档通常包含大量截图、示意图和图表。v0plex 通过 `dev/assets/` 目录管理这些资源。

在 Markdown 中引用图像时，使用相对路径。例如，位于 `dev/_01_指南/` 下的文档要引用 `dev/assets/images/` 中的图片，路径应写为 `../../assets/images/screenshot.png`。构建过程中，v0plex 会自动将这些图像复制到 `public/vmdimage/` 目录，并在最终的静态站点中正确引用。

图像文件命名建议使用小写字母、数字和连字符，避免使用空格和特殊字符。虽然 v0plex 支持带空格的文件名，但空格在 URL 中需要编码，会增加不必要的麻烦。

## 理解构建流程

了解 v0plex 的构建流程有助于在出现问题时进行诊断。完整的构建分为四个阶段。

第一阶段是清理。构建开始前，`vmd_parser/cleaner.ts` 会删除上一次构建的输出，包括 `app/vmdcode/`、`app/vmdjson/` 和 `public/vmdimage/` 目录。这确保了不会残留已删除内容的旧文件。

第二阶段是 Markdown 解析。`vmd_parser/main.ts` 遍历 `dev/` 目录，读取每个 Markdown 文件，解析 frontmatter，将 Markdown 内容编译为 TSX 代码。这个阶段会检测语法错误，比如未闭合的自定义标签、无效的嵌套结构等。所有错误都会显示在终端中，并指出出错的文件和行号。

第三阶段是搜索索引生成。`public/lex.py` 读取解析后的内容，使用 jieba 库对中文文本进行分词，生成倒排索引文件 `public/lexicon.json`。这个索引支持文档的全文搜索功能。

第四阶段是静态导出。Next.js 读取生成的 TSX 组件，渲染为 HTML 文件，输出到 `gh-page-output/` 目录。这个目录包含了部署站点所需的全部文件，可以直接上传到任何静态托管服务。

## 构建生产版本

当内容准备就绪，需要部署到生产环境时，执行：

```bash
pnpm build
```

这会执行与开发模式相同的解析和索引步骤，但使用 Next.js 的生产模式进行构建，生成的 HTML 会经过优化，包括代码压缩、资源哈希等。构建完成后，`gh-page-output/` 目录包含了完整的静态站点。

构建过程的输出会显示在终端中。如果一切正常，最后几行应该显示构建的页面数量和总耗时。如果有错误，需要根据提示修复问题后重新构建。

## 调试技巧

开发过程中可能遇到各种问题。以下是一些常见情况的排查方法。

如果页面显示空白或报错，首先检查终端是否有构建错误。v0plex 的错误报告通常很详细，会指出问题的文件和行号。修复后重新启动开发服务器。

如果导航树没有正确显示，检查文件名是否符合 `_NN_` 前缀的格式。数字不足两位需要补零，比如 `_01_` 而不是 `_1_`。

如果图像无法显示，确认图像路径是否正确。在开发模式下，v0plex 会验证图像文件是否存在，如果引用了不存在的文件，构建时会报错。

如果中文搜索没有结果，检查 `pnpm lex:gen` 是否成功执行。这个步骤依赖 Python 环境，如果虚拟环境有问题，可能会静默失败。可以单独运行 `python public/lex.py` 查看详细的错误信息。
