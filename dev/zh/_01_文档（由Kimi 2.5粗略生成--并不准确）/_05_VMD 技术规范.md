# VMD 技术规范

VMD（v0plex Markdown）是 v0plex 对标准 Markdown 的扩展集合，旨在解决技术文档中常见的布局需求。本章详细说明每个 VMD 标签的语法规则、约束条件和使用场景。

## 设计原则

VMD 的设计遵循几个核心原则。首先，扩展应该是声明式的而非命令式的，作者描述内容是什么，而不是如何呈现。其次，语法应该与标准 Markdown 保持一致性，使用类似的标记风格。第三，错误处理应该是严格且明确的，构建时发现问题比运行时出现问题更容易修复。最后，扩展应该是可组合的，不同的标签可以在合理范围内嵌套使用。

基于这些原则，VMD 采用了类似 HTML 的标签语法，但限制了可用标签的范围，并定义了严格的嵌套规则。这种设计使得 VMD 既保持了表达力，又避免了无限制的 HTML 带来的复杂性和安全风险。

## post 标签：并排布局

`<post>` 标签是 VMD 中最复杂的组件，它实现了左右并排的内容布局，左侧显示文本内容，右侧显示图像列表。这种布局在技术文档中非常常见，例如教程中的步骤说明配图、API 文档中的代码示例截图等。

### 语法结构

```markdown
<post>
<lft>

左侧内容，支持标准 Markdown 语法。
可以是段落、列表、代码等。

</lft>
<rt>

![图1](../../assets/images/a.png)
![图2](../../assets/images/b.png)

</rt>
</post>
```

`<post>` 标签必须包含且仅包含一个 `<lft>` 子标签和一个 `<rt>` 子标签，顺序固定为左在前、右在后。子标签之间必须有且仅有一个空格或换行分隔。

### lft 内容约束

`<lft>` 标签内的内容支持标准 Markdown 的大部分行内和块级元素，但出于布局稳定性的考虑，排除了以下元素：

代码块（三个反引号包围的代码）不允许使用，因为代码块的高度难以控制，可能破坏并排布局的视觉平衡。如果需要展示代码，建议使用行内代码或考虑其他布局方式。

块级数学公式（`$$...$$`）不允许使用，原因与代码块类似。行内数学公式 `$...$` 可以使用。

图像（`![alt](src)`）不允许使用，这是 `<lft>` 和 `<rt>` 的明确分工，左侧负责文本，右侧负责图像。

引用块（`> ...`）不允许使用，引用块通常带有特定的视觉样式（如左侧边框），可能与 post 容器的样式冲突。

其他 VMD 自定义标签（`info`、`warning`、`success`、`smallimg`）不允许在 `<lft>` 内嵌套使用。

此外，`<lft>` 内容的开头和结尾必须各有一个空行。这意味着 `<lft>` 标签后必须紧跟两个换行符（即一个空行），内容开始前也必须有两个换行符。

### rt 内容约束

`<rt>` 标签内的内容限制更为严格，只允许包含 Markdown 图像语法。每个图像必须独占一行，图像之间用空行分隔。图像必须包含替代文本（alt text），这是可访问性的基本要求。

```markdown
<rt>

![正确的图像描述](../../assets/images/correct.png)

![另一个图像](../../assets/images/another.png)

</rt>
```

与 `<lft>` 相同，`<rt>` 内容的开头和结尾也必须各有一个空行。

### 常见错误

构建时，以下情况会触发错误：

`<lft>` 和 `<rt>` 的顺序错误或数量不对会报告格式错误。`<lft>` 中使用了禁止的元素（代码块、块级公式、图像、引用块）会报告内容类型错误。`<rt>` 中包含了非图像内容会报告无效内容错误。图像缺少替代文本会报告缺少 alt 属性错误。标签之间的空格或换行数量不对会报告格式错误。

## 信息提示框标签

VMD 提供了三种信息提示框标签，用于突出显示特定类型的内容：`<info>` 用于一般性提示信息，`<warning>` 用于警告或注意事项，`<success>` 用于成功提示或最佳实践。

### 语法结构

```markdown
<info>
这是一条提示信息，可以包含 **粗体**、*斜体* 等行内格式。
也可以包含多行内容。
</info>

<warning>
这是一条警告信息，提醒读者注意潜在的问题。
</warning>

<success>
这是一条成功提示，表示推荐的方案或已完成的状态。
</success>
```

这三个标签的语法结构完全相同：开标签、内容、闭标签。内容支持标准 Markdown 的行内格式（粗体、斜体、行内代码、链接等）和块级格式（段落、列表、普通文本）。

### 内容约束

信息提示框标签的内容有以下限制：

不允许嵌套其他 VMD 自定义标签（包括其他 info、warning、success 标签，以及 post、smallimg 标签）。这种限制避免了视觉层次的混乱和样式冲突。

不允许包含代码块和块级数学公式，原因与 post 标签中的限制相同：这些元素的高度难以预测，可能破坏提示框的视觉呈现。

内容前后不需要额外的空行，标签直接包裹内容即可。但出于可读性考虑，建议在标签内使用空行分隔不同的段落。

### 视觉呈现

三种提示框在视觉上通过颜色和图标区分。info 通常使用蓝色系，warning 使用橙色系，success 使用绿色系。这种颜色编码遵循通用的设计惯例，读者可以迅速识别信息的类型和重要性。

## smallimg 标签：小图标注

`<smallimg>` 标签用于在段落中插入小尺寸的图像，常用于图标、缩略图或行内示例图像。与标准的 Markdown 图像语法不同，`<smallimg>` 渲染为固定的小尺寸，不会占据整行。

### 语法结构

```markdown
这是一段包含 <smallimg>![图标](../../assets/images/icon.png)</smallimg> 小图像的文本。
```

`<smallimg>` 是行内标签，必须在一行内开始和结束。标签内必须且只能包含一个 Markdown 图像语法，图像前后不能有空格或其他内容。

### 使用场景

`<smallimg>` 适用于以下场景：

在段落中插入图标或符号，例如操作系统的 logo、文件类型图标等。展示界面元素的缩略图，如按钮、输入框的截图。在步骤说明中插入小型的操作示意图。

不适合使用 `<smallimg>` 的情况包括：需要展示全尺寸截图、需要图像带有详细说明、图像本身包含重要文本信息。这些情况下应该使用标准的 Markdown 图像语法或 post 标签。

## 标签嵌套规则

VMD 标签的嵌套受到严格限制，以维护渲染的可预测性和视觉的一致性。理解这些规则有助于避免构建错误。

### 禁止的嵌套

以下嵌套方式会触发构建错误：

`<post>` 不能嵌套在其他任何 VMD 标签内。这意味着你不能在 info、warning、success 标签内使用 post 布局。

`<lft>` 和 `<rt>` 只能作为 `<post>` 的直接子标签出现，不能独立使用，也不能嵌套在其他标签内。

`<info>`、`<warning>`、`<success>` 之间不能相互嵌套，也不能嵌套在彼此内部。

所有 VMD 自定义标签都不能嵌套在 `<lft>` 内。

### 允许的嵌套

以下嵌套方式是允许的：

标准 Markdown 元素（段落、列表、强调等）可以嵌套在任何 VMD 标签内，只要符合该标签的内容约束。

`<smallimg>` 可以出现在标准 Markdown 段落中，也可以出现在 `<info>`、`<warning>`、`<success>` 的内容中。

列表可以嵌套在信息提示框内，这是组织多行提示内容的常用方式。

## 错误处理

v0plex 在构建阶段对 VMD 语法进行严格验证。任何语法错误都会阻止构建完成，并输出详细的错误信息。

### 错误报告格式

当检测到 VMD 语法错误时，构建工具会输出以下格式的信息：

```
ERROR [VMD-XXXX] Error description
  File: dev/_01_Example/_01_file.md
  Line: 25
  Details: Additional context about the error
```

错误代码（VMD-XXXX）用于唯一标识错误类型，便于搜索解决方案。文件路径和行号精确定位了错误发生的位置。详细信息提供了问题的具体描述和可能的修复建议。

### 常见错误代码

VMD-1001：EXTENSION_POST_INVALID_FORMAT — post 标签的格式不正确，可能是子标签顺序错误、数量不对，或子标签之间的分隔符不符合要求。

VMD-1002：EXTENSION_POST_LFT_CODE_BLOCK — 在 lft 中使用了代码块。解决方案是将代码改为行内格式，或考虑使用其他布局。

VMD-1003：EXTENSION_POST_LFT_BLOCK_MATH — 在 lft 中使用了块级数学公式。解决方案是使用行内数学公式或将公式移出 post 布局。

VMD-1004：EXTENSION_POST_LFT_IMAGE — 在 lft 中使用了图像。解决方案是将图像移到 rt 中，或移出 post 布局使用标准图像语法。

VMD-1005：EXTENSION_POST_LFT_BLOCKQUOTE — 在 lft 中使用了引用块。解决方案是改写为普通段落或列表。

VMD-1006：EXTENSION_POST_RT_INVALID_SPACING — rt 内容的间距不正确。检查 rt 标签后是否有空行，图像之间是否正确分隔。

VMD-1007：EXTENSION_POST_INVALID_CONTENT — rt 中包含了非图像内容。rt 只能包含图像，不能包含文本或其他 Markdown 元素。

## 性能考虑

VMD 标签的解析和渲染会带来一定的性能开销，但在正常使用场景下这些开销可以忽略不计。

解析阶段，每个 VMD 标签需要正则表达式匹配和字符串处理，复杂度与文档长度线性相关。对于包含大量 post 布局的文档，构建时间可能略有增加。

渲染阶段，复杂的布局（如 post）需要更多的 DOM 节点和 CSS 计算。在移动设备上，频繁的布局变化可能影响滚动性能，但 v0plex 的响应式设计已经对此进行了优化。

建议的使用限制：单个文档中 post 标签的数量不宜超过 20 个，过多的并排布局会使页面显得杂乱。rt 中的图像数量建议控制在 5 个以内，过多的图像会导致右侧栏过长，破坏视觉平衡。
