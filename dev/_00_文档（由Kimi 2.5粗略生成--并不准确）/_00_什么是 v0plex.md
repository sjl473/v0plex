# 什么是 v0plex

## 概述

v0plex 是一个面向技术文档创作的静态站点生成器。它的核心目标是将 Markdown 文件转换为结构化的静态网页，同时保持内容与技术实现的清晰分离。v0plex 的设计哲学强调内容的可维护性、构建流程的可靠性，以及最终呈现的精确控制。

与常规的静态站点生成器不同，v0plex 引入了 VMD（v0plex Markdown）的概念——一种扩展了标准 Markdown 语法的标记语言。这种扩展并非为了添加花哨的视觉效果，而是为了解决技术文档中常见的布局难题：并排的内容与图像、信息提示框、数学公式展示等。通过这些扩展，v0plex 使得作者能够用纯文本描述复杂的页面结构，而无需深入前端开发的细节。

## 架构设计

v0plex 采用三层架构来组织整个生成流程。这种分层方式确保了系统的可扩展性，也使得各个组件的职责边界清晰明确。

### 内容层（Content Layer）

内容层是 v0plex 系统的数据源头，所有需要发布的内容都存储在 `dev/` 目录中。这里的文件组织遵循特定的命名约定：文件夹和文件名以 `_NN_` 前缀开头，其中 `NN` 是两位数字，用于控制内容在导航树中的排序。这种约定使得作者能够通过文件系统直接管理内容的层级关系，而无需依赖额外的配置文件。

内容层中的每个 Markdown 文件都包含 YAML 格式的 frontmatter，用于定义文档的元数据：标题、创建时间、最后更新时间、作者等。这些元数据在构建过程中被提取并用于生成导航、页面头部信息等结构元素。

### 构建层（Build Layer）

构建层位于 `vmd_parser/` 目录，负责将原始内容转换为可供展示层消费的格式。这是一个用 TypeScript 编写的解析器，其工作流程包括：扫描内容层目录结构、解析 frontmatter 元数据、将 Markdown 编译为 TSX 组件代码、处理并复制图像资源，以及生成导航树数据。

构建层的核心组件是 MarkdownCompiler，它基于 Marked 库进行扩展，添加了 VMD 特有的语法解析器。这些解析器处理自定义标签（如 `<post>`、`<info>`、`<warning>` 等），将它们转换为特定的 React 组件调用。构建过程中，任何语法错误都会被捕获并以结构化的方式报告，包括错误类型、发生位置和建议的修复方案。

### 展示层（Presentation Layer）

展示层是一个基于 Next.js 的 React 应用程序，负责渲染最终的静态页面。它从构建层导入生成的 TSX 组件，并使用自定义的 VMD 组件库（位于 `components/vmd/`）来呈现内容。

展示层的视觉设计基于 IBM Carbon Design System，这使得生成的站点具有一致、专业的视觉风格。Carbon 的设计原则强调清晰、简洁和功能性，这与技术文档的需求高度契合。展示层同时支持亮色和暗色主题，读者可以根据偏好自由切换。

## 工作流程

v0plex 的完整工作流程涵盖了从内容编写到站点发布的全过程。首先，作者在 `dev/` 目录中编写 Markdown 文件，使用标准 Markdown 语法和 VMD 扩展标签来组织内容。随后，执行 `pnpm vmd:gen` 命令触发构建流程，解析器会扫描所有 Markdown 文件，生成对应的 TSX 组件和站点数据结构。接着，`pnpm lex:gen` 命令为中文内容生成搜索索引，这依赖 Python 的 jieba 分词库来实现中文文本的切分。最后，`pnpm build` 命令调用 Next.js 生成静态 HTML 文件，这些文件可以直接部署到任何静态托管服务。

这种工作流程的优势在于每一步都有明确的输出和验证机制。构建过程中的错误会被立即报告，而不是在部署后才暴露问题。同时，生成的站点是完全静态的，这意味着它可以在任何支持静态文件的托管服务上运行，无需服务器端渲染或数据库支持。

## 技术选型

v0plex 的技术栈经过精心选择，以平衡开发效率、运行性能和长期维护性。Next.js 14 作为基础框架，提供了优秀的静态导出能力和现代化的开发体验。TypeScript 的使用确保了代码的类型安全，这在大型项目中尤为重要。IBM Carbon Design System 提供了经过验证的设计模式和组件库，使得视觉一致性得到保障。Marked 作为 Markdown 解析引擎，以其性能和可扩展性著称，这为 VMD 语法的实现提供了坚实基础。

对于中文搜索，v0plex 选择了 jieba 分词库，这是中文自然语言处理领域广泛使用的工具，能够准确识别中文词语边界，从而提供高质量的搜索结果。这种混合使用 Node.js 和 Python 的方案，充分发挥了各自生态的优势，而不是强行在单一技术栈中解决所有问题。
